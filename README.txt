# WorkProject

A projektet alőször azzal kezdtem, hogy néztem egy videó sorozatot a github kezeléséről,
majd a reactal ismerkedtem, eddig sima html+css+javascriptes oldalakkal volt dolgom, szóval
ez sem volt sokkal különbözőbb de azért volt jó pár újdonság.

A bored api dokumentációjában minden feladatra találtam megoldást, viszont először nem tudtam mik 
azok a rest api-k és, hogy működnek, de egy két videó után megértettem és tudtam folytatni a munkát.

A tényleges kódolási folyamat úgy nézett ki, hogy elkezdem a feladatot -> megakadok -> videó -> próbálom használni
a videóból tanultakat -> debuggolás -> következő feladat.

A projekt külalakjával nem törődtem túl sokat, ha már nem ez volt a lényeg, inkább a feladatra koncentráltam.

Feladatok közül legtöbb időt a második feladat vitte, sokáig nem akart csak 1 activityt kiírni, vagy kiírt 10-et csak
egymás után gyorsan. Először minden feldadatot az App.js oldalon valósítottam meg, majd szétszedtem őket külön oldalra.

Összesszes 3 napot töltöttem a feladattal, kezdtem pénteken, akkor foglalkoztam vele 10 órától este 19 óráig, ebben volt 
pár ebédszünet, és sok tutorial videó nézése. Majd hétfőn kezdtem neki újra 17 órától és csináltam estig, 
a nap végére kész volt 4ből 2 feladat teljesen, a maradék 2 pedig hibásan működött. Kedden szétszetdem a feladatokat 
külön oldalakra, és befejeztem a feladatokat. Majd a dokumentációt befejeztem 16 órakor.

Az alkalmazást úgy kell futtatni, hogy nyitunk egy git bash-t a "workproject" mappában és "npm-start" parancsal megnyílik 
az alkalmazás az alapértelmezett böngészőnkben.
